Root cause analysis
What went wrong — and how to fix it
Every manual edit is a signal. Two root causes account for the majority of corrections.
Same product, different size unit — created as two separate groups
4 groups
What happened: 4 groups were merged by editors because the same product was stored twice: once as "1 kg" and once as "1000 g". The system read these as two different sizes and created two groups. Editors had to manually pull them back together.
Fix: Normalise all size values to a single unit (grams for weight, ml for volume) before grouping runs. This is a one-time data fix and would eliminate the majority of corrections in AP.

Different product formats put in the same group
19 groups
What happened: 19 groups were reduced to a single product after editors removed everything else. The removed products shared the same brand and category but were a different format — for example, a powder and a liquid, or a sachet and a bottle. The system cannot tell these apart today.
Fix: Scan product names for format keywords (powder, liquid, sachet, capsule, spray, tablet) and treat them as hard separators — two products with different format keywords should never be in the same group, even if brand and category match.

The honest hard case · 4% · 270 groups
When one group is missing a dimension the other has
Two groups can only merge if they split on the same axes. In these 270 cases one side splits on an attribute (here, Flavour) the other never set. To merge, we have to find that missing value and write it onto every member that lacks it

This happens in both directions. Sometimes our group carries the extra dimension and the live members lack it, so we enrich the existing side. Sometimes the live group carries it and our new members lack it, so we enrich our side. The work is the same either way: locate the missing value, and decide per product whether it can be assigned safely or needs a human. The attribute is usually Flavour, but the same pattern appears for any axis (sub-line, format, pack).

3

The decision mechanism, one call per product that lacks the value

Every blank from section 2 runs through one of these three outcomes.
Readable → enrich

The missing value is in the title (Chicken, Garlic, Full Cream) or safely inferable (an unmarked Heinz ketchup is Original). Assign it, flag any inferred value as assumed, and merge the two groups.

Ambiguous → review

The missing value is not readable, or naming it would introduce a variety the group has not seen (the Vegan ‘Nduja paste). Sent to a human queue to confirm the value before it is written. No silent guess.

Does not fit → split

The product does not really have the other group's dimension (a multi-flavour assortment, a different sub-line). Forcing a value would pollute the group, so it becomes its own clean, non-duplicate group.

Learnings
Highlights
  • 97.7% group acceptance rate across 2,392 reviewed groups (Foodpanda + HungerStation)
  • Model unlocked variant grouping for HungerStation and PedidosYa at meaningful scale for the first time — from ~1K to 19K and 17K groups
  • Quality held on a new platform: HungerStation had zero broken groups
Lowlights
  • Two root causes drove all 55 corrections: unit inconsistency (1 kg vs 1000 g) and format confusion (powder vs liquid, sachet vs bottle)
  • 19 groups broken — all Foodpanda APAC, all format confusion
  • 95.4% of groups never reviewed — bulk quality still unknown
  • iGMV not fully measurable — reporting bugs on PedidosYa and Foodpanda
Systemic Insight
  • Errors surfaced from mismatches in product types, master categories, and brand IDs. The grouping contract and I/O format were updated accordingly.
  • Legacy files kept temporarily for cleanup and platform communications — not for new runs

Root Cause 1 — Unit inconsistency (4 groups, Foodpanda APAC)
The same physical product stored twice as "1 kg" and "1000 g" — treated as two different sizes, creating duplicate groups. Examples: Topvalu Tea, Little Sheep Hot Pot Base, mitr phol Sugar, Ready Seafood.

Root Cause 2 — Format confusion (19 groups, Foodpanda APAC — all broken)
Products sharing the same brand and category but different formats placed in the same group. Top categories: Coffee (7), Instant Coffee (3), Baking Needs (3), Tea (3), Hot Pot Base (2), Pasta Sauce (2), Soy Sauce (2), Body Care (2).
Examples: Cheerios Multi Grain vs Frosted variants; Nescafe 3-in-1 sachets vs instant jars; Hai Di Lao mushroom vs spicy hotpot bases.
